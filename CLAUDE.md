# Struq

## Overview

Struq is a web app for musicians to quickly map out song structures (Intro ‚Üí A ‚Üí B ‚Üí Chorus ‚Üí Solo ‚Üí Outro) and reference them during practice or performance.

**URL:** struq.focuswave.cc
**AI:** Google Gemini Flash (song structure generation from prompt)

## Project Layout

- `src/routes/` ‚Äî TanStack Start file-based routing (`routeTree.gen.ts` is generated)
- `src/auth/` ‚Äî Google OAuth, sessions, `requireUser()` wiring
- `src/db/` ‚Äî Drizzle schema + `getDb()` factory (binding name `DB`)
- `src/server/` ‚Äî shared server helpers (`requireUser`, `now`)
- `src/songs/`, `src/setlists/` ‚Äî feature server-fns + components
- `src/offline/` ‚Äî PWA: SW registration, IndexedDB cache (idb), bulk download, offline error boundary
- `src/i18n/` ‚Äî `ja` / `en` locales + `I18nProvider`
- `src/ui/` ‚Äî Broadcast Console primitives (top-bar, side-rail, structure-bar, tokens, icons)
- `src/lib/` ‚Äî cross-cutting utilities (logger, schemas, rate-limit, theme)
- `drizzle/` ‚Äî D1 migration SQL files

## UI/UX Guidelines

- **Broadcast Console** design language: dark ink surfaces (`--color-ink*`), high-contrast mono labels (JetBrains Mono), subtle line dividers (`--color-line*`)
- Sections color-coded via OKLCH palette (`--color-section-*`); section A (blue) is the app-wide accent (`C.accent`), chorus amber is reserved for chorus only
- Touch-first design (large tap targets); `touch-landscape` variant = landscape on touch devices only (excludes desktop)
- Performance view optimized for iPad landscape (fullscreen, dark)
- Edit/list views optimized for mobile (iPhone)
- Animations kept subtle: `led-pulse`, `live-pulse`, `beat-pop` defined in `src/styles.css`

## Environment Variables

For local dev, create `.dev.vars` (NOT `.env`) ‚Äî Wrangler injects this into the Worker context during `pnpm dev`. For production, use `wrangler secret put`.

Required: `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`, `GEMINI_API_KEY`, `SESSION_SECRET` (‚â•32 chars).

## Gotchas

- Husky `pre-commit` runs `pnpm typecheck` + `lint-staged` (biome auto-fix); type errors block the commit ‚Äî fix and re-stage rather than `--no-verify`
- E2E auth state is stored in `e2e/.auth/user.json` (gitignored); global setup hits `/api/auth/test-login`
- Server functions use `createServerFn()` + `.inputValidator(ZodSchema)` + `requireUser()` ‚Äî see any `*server-fns.ts` for reference
- `src/routeTree.gen.ts` is auto-generated by TanStack Start ‚Äî do not edit manually
- D1 binding name is `DB` (defined in `wrangler.jsonc`)
- `@/*` resolves to `./src/*`
- Soft deletes via `deleted_at`; exception: SetlistSongs uses hard delete
- D1 has a 100 bound-parameter limit per statement; section inserts batch at 10 rows (9 columns √ó 10 = 90 params) to stay under the limit
- Sections `label` field is only used for `custom` type ‚Äî fixed types derive their label from locale at render time
- Unit tests cover pure functions only (schemas, validation, constants); server functions depend on D1 + session and are tested via E2E only
- Server functions that operate on both a setlist and songs must verify ownership of **both** (see `addSongToSetlist` for the reference pattern)
- Design tokens: use `import { C } from "@/ui/tokens"` for dynamic inline styles (per-section colors, box-shadow); for static Tailwind utilities use arbitrary values like `bg-[color:var(--color-ink)]`
- Legacy tokens (`--color-surface*`, `--color-text-primary/secondary/on-dark`) coexist with Broadcast Console tokens ‚Äî they're deprecated per-screen as each redesign PR lands; prefer `--color-ink*` / `--color-text` / `--color-dim*` for new code
- Offline support uses `vite-plugin-pwa` + service worker + IndexedDB (`src/offline/db.ts`); cache schema is versioned ‚Äî bump the version on shape changes and ensure logout wipes the IDB store
- `AGENTS.md` is a symlink to `CLAUDE.md` (consumed by other agentic tools); edit `CLAUDE.md` only ‚Äî never replace `AGENTS.md` with a regular file
